# 🔐 로컬 개발 환경 - 임시 로그인 설정 가이드

로컬 환경에서 3개의 테스트 계정으로 로그인하여 모든 기능을 테스트할 수 있습니다.

---

## 📋 테스트 계정

| 역할 | 아이디 | 비밀번호 | 설명 |
|------|--------|----------|------|
| 조사원(청소자) | `cleaner` | `1234` | 현장에서 쓰레기 수거 데이터 입력 |
| 운반자 | `driver` | `1234` | 수거 지점 확인 및 수거 작업 |
| 관리자 | `admin` | `1234` | 전체 데이터 관리 및 분석 |

---

## 🚀 로컬 로그인 사용 방법

### 1단계: 프로젝트 실행

```bash
# 프로젝트 디렉토리에서
pnpm dev
```

### 2단계: 브라우저 접속

```
http://localhost:3000
```

### 3단계: 로그인

**방법 1: 빠른 로그인 (권장)**
1. 홈페이지에서 "로컬 로그인" 버튼 클릭
2. 원하는 테스트 계정 선택 (조사원/운반자/관리자)
3. 자동으로 로그인됨

**방법 2: 수동 입력**
1. 홈페이지에서 "로컬 로그인" 버튼 클릭
2. 아이디와 비밀번호 수동 입력
3. "로그인" 버튼 클릭

---

## 🎯 각 역할별 테스트 시나리오

### 조사원(청소자) 테스트

**계정**: `cleaner` / `1234`

1. 로그인 후 홈페이지에서 "조사원 페이지" 클릭
2. 필수 정보 입력:
   - 해안명: "테스트 해안"
   - 위치: 자동 또는 수동 입력
   - 수거량: 5 (마대 50L 기준)
   - 해안 길이: 100 (미터)
   - 주요 쓰레기: 2-3개 선택
3. 쓰레기가 있는 사진 2장 업로드 (청소 전/후)
4. "보고서 전송" 클릭
5. 완료 메시지 확인

### 운반자 테스트

**계정**: `driver` / `1234`

1. 로그인 후 홈페이지에서 "운반자 페이지" 클릭
2. 지도에 표시된 수거 지점 확인
3. 수거 목록에서 항목 선택
4. 집하장 선택 (예: 해운대 집하장)
5. 예상 운송량 및 이동 거리 확인
6. "수거 완료" 버튼 클릭

### 관리자 테스트

**계정**: `admin` / `1234`

1. 로그인 후 홈페이지에서 "관리자 페이지" 클릭
2. 통합 대시보드 확인:
   - 지도에 수거 지점 표시
   - 쓰레기 종류별 분포 (막대그래프)
3. 필터 드롭다운에서 특정 쓰레기 종류 선택
4. 보고서 클릭 시 AI 탐지 결과 확인:
   - 청소 전 사진 위에 탐지 객체 표시
   - 객체 종류 및 신뢰도 표시
5. 주요 통계 그래프 확인

## 🔧 로컬 로그인 작동 원리

### 저장 위치

로그인 정보는 **브라우저 로컬 스토리지**에 저장됩니다:

```javascript
// 저장되는 데이터
localStorage.getItem("local_user")
// {
//   "id": "random-id",
//   "openId": "local-cleaner",
//   "name": "조사원(청소자)",
//   "email": "cleaner@local.dev",
//   "role": "user",
//   "loginMethod": "local"
// }
```

### 로컬 모드 감지

시스템은 다음 호스트에서 자동으로 로컬 모드를 활성화합니다:
- `localhost`
- `127.0.0.1`
- `192.168.x.x` (로컬 네트워크)

## 🔄 로그인 상태 관리

### 로그인 유지

새로고침해도 로그인 상태가 유지됩니다 (로컬 스토리지 사용).

### 로그아웃

1. 홈페이지 우측 상단의 "로그아웃" 버튼 클릭
2. 로컬 스토리지에서 사용자 정보 삭제
3. 로그인 페이지로 이동

### 계정 전환

1. "로그아웃" 클릭
2. 다른 계정으로 "로컬 로그인" 클릭
3. 새 계정으로 로그인

## 📝 데이터 저장

### 로컬 스토리지 데이터

로컬 로그인으로 생성된 데이터는 **SQLite 데이터베이스**에 저장됩니다:

```bash
# 데이터베이스 확인
sqlite3 local.db

# 쿼리 예시:
sqlite> SELECT * FROM users;
sqlite> SELECT * FROM reports;
sqlite> SELECT * FROM ai_detections;
```

### 데이터 초기화

```bash
# 데이터베이스 초기화
rm local.db
pnpm db:push
```

## 🆘 문제 해결

### 문제 1: 로컬 로그인 버튼이 보이지 않음

**원인**: 로컬 모드가 감지되지 않음

**해결책**:
```bash
# localhost에서 접속 확인
http://localhost:3000

# 또는 127.0.0.1 사용
http://127.0.0.1:3000
```

### 문제 2: 로그인 후 페이지가 로드되지 않음

**원인**: 브라우저 캐시 문제

**해결책**:
```bash
# 브라우저 DevTools 열기 (F12)
# Application → Local Storage → 모든 항목 삭제
# 페이지 새로고침 (Ctrl+Shift+R)
```

### 문제 3: "아이디 또는 비밀번호가 잘못되었습니다" 오류

**해결책**:
- 아이디 정확히 입력: `cleaner`, `driver`, `admin`
- 비밀번호 정확히 입력: `1234`
- 대소문자 구분 없음

### 문제 4: 로그인 후 데이터가 저장되지 않음

**원인**: 데이터베이스 연결 오류

**해결책**:
```bash
# 데이터베이스 마이그레이션 실행
pnpm db:push

# 또는 수동으로
drizzle-kit generate
drizzle-kit migrate
```

## 📚 관련 파일

로컬 로그인 기능은 다음 파일에서 구현됩니다:

| 파일 | 설명 |

| `client/src/_core/hooks/useLocalAuth.ts` | 로컬 인증 훅 |

| `client/src/App.tsx` | 로컬 로그인 라우트 등록 |

## 🔐 보안 참고사항

⚠️ **중요**: 로컬 로그인은 **로컬 개발 환경용**입니다.
